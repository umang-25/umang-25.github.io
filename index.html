Introduction

This app simply tells you the different specifications available for you car model if you provide it the manufacturer company's name, model and the year of manufacture. This project could be easily scaled to work on bigger datasets and solve the problem of finding the different models available for your car and thus help in deciding the best car you could buy.

What is it based on ?

It uses the mpg dataset. the mpg dataset contain a subset of the fuel economy data that the EPA makes available on http://fueleconomy.gov. It contains only models which had a new release every year between 1999 and 2008 - this was used as a proxy for the popularity of the car. This dataset was used because of its realtively small size and the popular cars used in the dataset.

What are the inputs and outputs

You'll simply input the manufacturer of the car say.. "audi" , a model you want to inquire about.. say "a4", and the year of manufacture "1999". The program will return:

  • 1.) Total models available
  • 2.) Different fuels model available say P for petrol or E for electronic
  • 3.) Different types of transmission models available
  • 4.) What are the Highway per Gallon variants available
  • 5.) Different types of cars available

What this Project does behind the UI ?

The project takes the input from the user then loads the mpg dataset and using dplyr subsets the models that match all the specifications. This is done in a reactive statement because then this subsetted dataset is then used return the rows and various models. The overall idea can be demonstrated here

                library(dplyr)
                mpg <- read.csv('mpg.csv')
                # assuming that manufacturer is audi, model is a4 and year of manufacture is 1999
                mo<- mpg %>% filter(manufacturer=="audi",model=="a4",year==1999)
                print("Total models available:")

## [1] "Total models available:"

                nrow(mo)

## [1] 4
